Start to Scale Workbook

Design Framework

Core Design Principles

Self-contained with bridges

Each chapter stands alone. No dependencies that block progress—only bridges to relevant chapters.

Concrete deliverables

One tangible, shareable artifact per chapter and phase. Users leave with something they can use immediately.

Modular design

Supports sequential reading or jumping to specific chapters. Users work where they need to.

Target User Profile

Primary

Founder or operator scaling a company.

Also serves

Leadership teams, scale-up facilitators, solo founders.

Not for

Passive readers, academic audiences.

Using the Workbook

Each phase can stand on its own.

Each chapter can stand on its own.

Completing all chapters in a phase delivers maximum value.

The Architecture: Three Chapter Types

Diagnostic

When to use:

Identifying what's broken or where to focus

Examples: Ch 1 (Barriers), Ch 2 (Right People/Right Things), Ch 10 (Cash)

Template

When to use:

Creating a specific artifact or plan

Examples: Ch 3 (Core Values), Ch 6 (OPSP), Ch 8 (Quarterly Theme)

Framework

When to use:

Applying a methodology to current situation

Examples: Ch 4 (De-Hassle), Ch 5 (Brand Promise), Ch 7 (Top 1 of 5)

Chapter Patterns by Type

All chapters share the same bookends: Opening → [Unique Middle] → Bridge

Diagnostic

  1. Opening
  2. Concept Summary3-4 key concepts
  3. Universal AssessmentQuick triage with explicit filter
  4. Conditional Deep DivesModular, self-contained sections (5-10 min each)
  5. Action PlanSynthesizes completed deep dives only
  6. Contextual Bridges

Template

  1. Opening
  2. Concept SummaryFramework explanation
  3. Template/WorksheetStep-by-step completion with examples
  4. Bridge

Framework

  1. Opening
  2. Framework ExplanationModel with real examples
  3. Application ExerciseApply to user's situation
  4. Output/Next ActionWhat to do with completed work
  5. Bridge
