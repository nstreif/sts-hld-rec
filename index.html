Design Framework Review

Core Design Principles

Self-contained with bridges

Each chapter and phase stands alone with bridges to other relevant phases, not dependencies that block progress.

Concrete deliverables

One concrete, shareable deliverable per chapter and phase.

Modular design

Supports sequential reading or jumping to specific chapters. Users work where they need to.

Target User Profile

Primary

Founder or operator scaling a company (assume working through the workbook independently).

Also serves

Leadership teams, solo founders, people working with STS Coaches

Not for

Passive readers, academic audiences.

Using the Workbook

Each phase can stand on its own.

Each chapter can stand on its own.

Completing all chapters in a phase delivers maximum value.

Appendix: The Architecture

Three Chapter Types

Diagnostic

When to use:

Identifying what's broken or where to focus

Examples: Ch 1 (Barriers), Ch 2 (Right People/Right Things), Ch 10 (Cash)

Template

When to use:

Creating a specific artifact or plan

Examples: Ch 3 (Core Values), Ch 6 (OPSP), Ch 8 (Quarterly Theme)

Framework

When to use:

Applying a methodology to current situation

Examples: Ch 4 (De-Hassle), Ch 5 (Brand Promise), Ch 7 (Top 1 of 5)

Appendix: Chapter Patterns by Type

All chapters share the same basic structure: Opening → Concept Summary → [Unique Middle] → Bridge

Diagnostic

  1. Opening
  2. Concept Summary: 3-4 key concepts
  3. Universal Assessment: Quick triage with explicit filter
  4. Conditional Deep Dives: Modular, self-contained sections (5-10 min each)
  5. Action Plan: Synthesizes completed deep dives only
  6. Bridge

Template

  1. Opening
  2. Concept Summary: 3-4 key concepts
  3. Template/Worksheet: Step-by-step completion with examples
  4. Bridge

Framework

  1. Opening
  2. Concept Summary: 3-4 key concepts + real examples
  3. Application Exercise: Apply to user's situation
  4. Output/Next Action: What to do with completed work
  5. Bridge
